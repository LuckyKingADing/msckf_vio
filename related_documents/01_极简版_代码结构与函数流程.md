# MSCKF-VIO 极简版：代码结构与函数流程

> 基于论文 *"Robust Stereo Visual Inertial Odometry for Fast Autonomous Flight"* (Sun et al., 2018) 实现的双目视觉惯性里程计。

---

## 一、系统概览

系统由**前端（ImageProcessor）**和**后端（MsckfVio）**两个 ROS Nodelet 组成：

```
双目图像 + IMU  →  前端（特征提取与跟踪）  →  CameraMeasurement  →  后端（EKF 滤波）  →  位姿输出
```

---

## 二、程序入口

两个 Nodelet 插件，由 launch 文件加载：

| 入口文件 | 入口函数 | 作用 |
|---------|---------|------|
| `image_processor_nodelet.cpp` | `ImageProcessorNodelet::onInit()` | 创建 `ImageProcessor` 实例并调用 `initialize()` |
| `msckf_vio_nodelet.cpp` | `MsckfVioNodelet::onInit()` | 创建 `MsckfVio` 实例并调用 `initialize()` |

---

## 三、前端流程（ImageProcessor）

### 初始化

```
onInit() → initialize()
    ├── loadParameters()       // 加载相机内参、外参、处理器参数
    ├── FastFeatureDetector::create()  // 创建 FAST 角点检测器
    └── createRosIO()          // 订阅双目图像+IMU，发布特征消息
```

### 核心回调：stereoCallback()

```
stereoCallback(cam0_img, cam1_img)
│
├── createImagePyramids()              // 构建 cam0/cam1 光流金字塔
│
├── [首帧] initializeFirstFrame()
│       ├── FAST 检测 cam0 特征点
│       ├── stereoMatch()              // cam0→cam1 双目匹配（LK光流 + 极线校验）
│       └── 按网格分配，每格保留响应值最高的特征
│
├── [后续帧]
│   ├── trackFeatures()                // 光流跟踪 + 外点剔除
│   │       ├── integrateImuData()     // IMU 积分 → 帧间旋转 R_p_c
│   │       ├── predictFeatureTracking()  // 用旋转预测特征初始位置
│   │       ├── calcOpticalFlowPyrLK()    // cam0 前帧→当前帧 LK 光流
│   │       ├── stereoMatch()             // cam0→cam1 双目匹配
│   │       ├── twoPointRansac() × 2      // cam0 帧间 + cam1 帧间 RANSAC
│   │       └── 双约束过滤，更新特征
│   ├── addNewFeatures()               // 在特征不足区域补充新 FAST 特征
│   └── pruneGridFeatures()            // 每格保留 lifetime 最长的 grid_max 个特征
│
├── publish()                          // 发布 CameraMeasurement（id, u0, v0, u1, v1）
└── 更新 prev ← curr
```

## 四、后端流程（MsckfVio）

### 初始化

```
onInit() → initialize()
    ├── loadParameters()           // 加载噪声参数、初始协方差、外参、滑窗大小等
    ├── 初始化连续噪声协方差 Q_c (12×12)
    ├── 初始化卡方检验表（自由度 1~99，置信度 0.95）
    └── createRosIO()              // 订阅 IMU+特征，发布里程计+点云
```

### 核心回调

**imuCallback()：**
```
imuCallback(msg)
    ├── 缓存到 imu_msg_buffer
    └── [前200帧] initializeGravityAndBias()  // 静止初始化重力方向和陀螺偏差
```

**featureCallback()（主循环）：**
```
featureCallback(msg)
│
├── 1. batchImuProcessing()           // IMU 状态传播（预测步）
│       └── processModel()            // 构建 F(21×21)、G(21×12)、Phi（3阶泰勒）
│           └── predictNewState()     // 四阶 Runge-Kutta 积分
│
├── 2. stateAugmentation()            // 新增相机状态 + 协方差扩展
│
├── 3. addFeatureObservations()        // 将当前帧观测加入特征管理器
│
├── 4. removeLostFeatures()            // 丢失特征 → 三角化 → EKF 更新
│       ├── checkMotion()              // 检查运动充分性
│       ├── initializePosition()       // LM 优化三角化
│       ├── featureJacobian()          // 雅可比 + 零空间投影
│       ├── gatingTest()               // 卡方门限检验
│       └── measurementUpdate()        // EKF 量测更新
│
├── 5. pruneCamStateBuffer()           // 裁剪滑动窗口
│       ├── findRedundantCamStates()   // 选择冗余帧
│       └── measurementUpdate()        // 对关联观测做 EKF 更新
│
├── 6. publish()                       // 发布 Odometry + TF + 特征点云
└── 7. onlineReset()                   // 位置方差超阈值则重置
```

## 五、核心数据结构

| 结构 | 维度 | 说明 |
|------|------|------|
| IMUState | 21维误差状态 | δθ(3) + b_g(3) + v(3) + b_a(3) + p(3) + 外参旋转(3) + 外参平移(3) |
| CAMState | 6维 | δθ_cam(3) + δp_cam(3) |
| Feature | — | id + observations(各帧观测) + 3D位置 |
| 协方差 P | (21+6N)×(21+6N) | N = 滑窗中相机状态数，最大 N=20 |

                       ──→  MsckfVio::imuCallback()

/cam0/image_raw  ──┐
/cam1/image_raw  ──┴──→  ImageProcessor::stereoCallback()

                         features (CameraMeasurement)
                                    │
                                    ▼
                         MsckfVio::featureCallback()
                                    │
                           ┌────────┴────────┐
                     vio/odom         vio/feature_point_cloud
