**MSCkF**
A Multi-State Constraint Kalman Filter for Vision-aided Inertial Navigation pdf文档阅读：
* 本文提出MSCKF（多状态约束卡尔曼滤波）算法，用于视觉辅助惯性导航，核心创新是构建不将 
  3D 特征位置纳入滤波状态向量的测量模型，通过多相机位姿几何约束实现位姿估计，计算复杂度随特征数量呈线性增长，仅与状态向量维度呈至多三次方关系；算法基于EKF设计，状态包含当前 IMU 状态与最多 Nmax 个历史相机位姿，经实测在 3.2km 轨迹下最终位置误差约 10m，相对误差0.31%，单核处理器可实现14Hz实时运行。
* MSCKF 是基于 EKF 的误差状态滤波，状态里只存 IMU + 最近 N 个相机位姿，不存 3D 路标点；通过多帧特征观测构建位姿约束，用左零空间投影消去路标，实现高效、高精度位姿估计。

论文阅读 Robust Stereo Visual Inertial Odometry for Fast Autonomous Flight
* stereo其实可以翻译成 双目? 翻译软件是翻译成立体
* 尤其对于微型飞行器 (MAV) 而言，正确的姿态估计对于保持机器人在空中的稳定至关重要,将来自摄像头的视觉信息与来自惯性测量单元 (IMU) 的测量数据相结合的解决方案，通常被称为视觉惯性里程计 (VIO)，因其在 GPS 信号受限的环境中也能良好运行而广受欢迎，并且与基于激光雷达的方法相比，它只需要一个小型轻便的传感器组件,因此，它是 MAV 平台的首选技术。
* VIO算法的挑战:例如光照条件剧烈变化、光照不均、低纹理场景以及由于阵风或剧烈机动导致的姿态突变。
* 所提出的立体视觉估计方法能够达到与最先进的单目解决方案相当甚至更高的效率(计算成本),
* 首个无需GPU加速即可在笔记本电脑上运行的开源基于滤波器的立体视觉信息采集（VIO）算法;对所提出的S-MSCKF算法与最先进的开源VIO解决方案（包括OKVIS [4]、ROVIO [5]和VINSMONO [6]）进行了详细的实验比较，比较指标包括精度、效率和鲁棒性
* 由于状态向量中四元数存在单位模长约束，直接使用真实 IMU 状态会导致协方差矩阵奇异。因此本文采用IMU 误差状态
> 为什么用误差状态而不用直接状态？四元数有单位约束（∥q∥=1），直接估计会让协方差矩阵奇异。误差状态用3 维小角度表示旋转，无冗余、无约束，适合卡尔曼滤波

* **B. Measurement Model 测量模型**
  * 虽然状态向量只包含左相机位姿，但右相机位姿可通过标定好的外参直接算出。
  * 如果双目图像已矫正，观测可降到 3 维。但本文用 4 维表示，好处是：不要求左右观测共面，因此不需要做双目矫正。3 维观测：必须双目矫正 → 左右图行对齐;4 维观测：不需要双目矫正 → 直接用原始左右图像素
  * 零空间投影 = 让不想影响的项变成 0
  * 测量模型 = 双目 4 维观测 + 无矫正 + 路标点最小二乘估计 + 线性化 + 零空间投影 → 标准 EKF 更新
* **C. Observability Constraint 观测约束**
  * 基于 EKF 的 6 自由度（6-DOF）运动估计 VIO存在4 个不可观测方向，对应全局位置（3 维）和沿重力轴的旋转（即偏航角 yaw，1 维）
    * 3 维全局位置：VIO 无法确定 “自己在世界的哪个绝对位置”（只能算相对运动）；
    * 1 维偏航角（yaw）：重力方向能约束 roll/pitch，但无法约束绕重力轴的旋转（比如无人机原地转方向，VIO 无法感知）；
  * 直接（朴素）实现 EKF-VIO 会让偏航角引入虚假信息（spurious information），这是因为：同一时刻下，过程模型与测量模型的线性化点不一致;过程模型（IMU 积分）的线性化点：基于上一时刻的估计值,测量模型（视觉观测）的线性化点：基于当前时刻的估计值,会导致滤波发散
  * 目前有多种维持滤波器一致性的方法:初值估计雅可比 EKF（FEJ-EKF）[23];可观测性约束 EKF（OC-EKF）[28];机器人中心映射滤波器（Robocentric Mapping Filter）[29]
    * 在本文实现中，选择应用 OC-EKF,原因:
      * 与 FEJ-EKF 不同，OC-EKF不高度依赖精准的初始估计值
      * 与 Robocentric Mapping Filter 相比，状态向量中的相机位姿可基于惯性系（而非最新 IMU 系）表示，因此在传播阶段，状态向量中已有相机状态的不确定性不会受最新 IMU 状态不确定性的影响;(相机状态的不确定性	可以理解为 “算法对这个相机位姿的‘怀疑程度’”—— 数值越大，越不确定)
* **D. Filter Update Mechanism 滤波更新机制**
  * 在我们的实现中，每隔一次更新步骤移除两个相机状态。在这两个相机状态上获取的所有特征观测都将用于测量更新。注意，由于上述原因，这两个相机状态上的双目观测只有在观测到同一个特征时才是有效的
  * 虽然频繁移除相机状态可能导致部分有效观测被忽略，但在实际实验中我们发现，即使使用更少的观测，估计性能也几乎不受影响
  * 为了选择要移除的两个相机状态，我们采用了类似于文献 [30] 提出的双向边缘化方法的关键帧选择策略。根据倒数第二个相机状态与其前一个相机状态之间的相对运动，选择移除倒数第二个相机状态或者最旧的相机状态。该选择过程执行两次，以选出两个要移除的相机状态。
  * 注意：最新的相机状态始终被保留，因为它包含新检测到的特征的观测
* **E. Image Processing Frontend 图像处理前端**
  * 选用 FAST 角点检测器[31]，核心原因是计算高效;已检测的特征点通过 KLT 光流算法[32] 进行时序跟踪（帧间匹配）
  * 尽管文献 [27] 指出：基于描述子的时序特征跟踪在精度上优于 KLT 方法;基于描述子的方法消耗 CPU 资源远更多，但精度提升微乎其微 → 不符合本文 “低算力、实时性” 的应用需求。
  * 本文同样用 KLT 光流完成双目特征匹配（而非描述子匹配），相比描述子方法进一步节省计算量。
  * 在 20cm 基线 的双目配置下，KLT 跟踪可稳定匹配深度 > 1m 的角点特征（跨双目图像）
  * 图像处理前端实现了两种外点剔除策略:2 点 RANSAC：移除时序跟踪中的外点;环形匹配（参考 [33]）：在 “前一帧 - 当前帧” 双目图像对间执行，进一步剔除特征跟踪、双目匹配步骤中产生的外点。
  * 图像处理前端 = FAST 检测 + KLT 光流（时序 + 双目） + 2 点 RANSAC + 环形匹配外点剔除 → 低算力、高实时、精度够用

Experiments
**A. EuRoC Dataset**
* 在非结构化、未知的环境中进行自主飞行，该环境包括仓库、林地和跑道
* 采集设备：无人机（MAV）搭载 VI 传感器，输出 20Hz 双目图像 + 200Hz IMU 数据（时间同步）。
* 数据集特点：旋转剧烈、光照变化大 → 对视觉惯性估计算法挑战大（能测出算法鲁棒性）
* 异常情况：S-MSCKF 在 V2_03_difficult 数据集失效 → 原因：本文用 KLT 光流做「时间特征跟踪 + 双目匹配」提升效率，但该数据集双目图像亮度持续不一致 → KLT 匹配失败 → 滤波发散。
* 测试硬件：NUC6i7KYK（四核 i7-6700HQ 2.6GHz）→ 无人机常用的嵌入式算力平台，结果有实际参考价值。
* 细节差异：
  * ① OKVIS 比 VINS-MONO 算力高 → 原因：用 Harris 角点 + BRISK 描述子（特征提取 / 匹配更耗算力），且后端尽可能快运行（VINS-MONO 后端固定 10Hz）；
  * ② S-MSCKF 算力分布：80% 消耗在前端（特征检测 / 跟踪 / 匹配），滤波本身仅占单核心 10% 算力（20Hz 运行）→ 说明算法瓶颈在视觉前端，滤波核心逻辑极轻量化
* RMS和CPU前后端力理统计方式：每个数据集跑 5 次取平均 → 结果更可靠；
* ROS 节点拆分：VINS-MONO/S-MSCKF 前端、后端分开运行 → 图表中浅色 = 前端算力，深色 = 后端算力

* **B. Fast Flight Dataset 快速飞行数据集**
  * 在机场跑道上采集的四组快速飞行数据集上对算法进行了评估，其最大速度分别为 5 m/s、10 m/s、15 m/s 和 17.5 m/s。在每组实验中，四旋翼无人机被指令飞往 300 m 外的航点并返回起点
  * 传感器配置包括：两台前视的 PointGrey CM3‑U3‑13Y3M‑CS 相机，以 40 Hz 频率(比EuRoC更高的频率)采集 960×800 分辨率图像；以及一台 VectorNav VN100 加固型 IMU，工作频率 200 Hz。整套传感器系统基于 IMU 触发信号实现时间同步。
  * 精度通过对齐时间与偏航角后，计算估计轨迹与 GPS 位置在 x、y 方向的 RMSE 得到。实验可以观察到：S‑MSCKF 在保持与其他方法相近精度的同时，实现了最低的 CPU 占用。
  * 值得注意的是，与 EuRoC 数据集上的实验相比，本文方法在图像处理前端花费了更多计算量。原因之一是图像频率与分辨率更高；另一原因是剧烈飞行导致图像剧烈变化,特征寿命更短，从而需要更频繁地检测新特征
* **C. Autonomous Flight in Unstructured Environments 非结构化环境中的自主飞行**
  * 飞行任务:树林穿行 → 找仓库入口 → 找目标 → 自主返航;全程无人干预，完全自主
  * 环境有多难？
    * 特征稀疏（feature poverty）：草地、墙面等纹理少，容易跟踪失败
    * 剧烈机动（aggressive maneuvers）：速度快、姿态变化大
    * 室内外切换：光照剧烈变化，视觉容易跟丢
  * 激光雷达 只用来建图
  * 定位（状态估计）只用：双目相机 + IMU（S-MSCKF）→ 纯视觉惯性定位，没有 GPS、不借助力雷达定位
  * 结果
    * 轨迹总长：约 700 米
    * 最终漂移：约 3 米
    * 相对误差：< 0.5%
    * 在长距离、复杂环境、强光照变化下，这个精度属于优秀水平。
  * 表明:在真实室外复杂环境、长距离、高速自主飞行中也能稳定、高精度工作，适合实际无人机部署
